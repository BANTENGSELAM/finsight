# Contributing to FinSight

Thank you for your interest in FinSight.

This project is primarily developed as a Software Engineering Portfolio and Case Study.

## Development Workflow

1. Create a feature branch.
2. Implement one feature only.
3. Follow MVC + Service Layer architecture.
4. Keep Controllers thin.
5. Place business logic inside Services.
6. Update documentation after each completed sprint.
7. Submit changes through a Pull Request.

---

## Coding Standards

- Follow PSR-12.
- Use meaningful variable names.
- Avoid duplicated code.
- Write clean and readable code.
- Prefer reusable components.

---

## Commit Convention

Examples:

feat: add expense management

fix: resolve validation issue

docs: update sprint documentation

refactor: improve transaction service

style: improve dashboard layout

test: add transaction service tests

---

Thank you for contributing.
